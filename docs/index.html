◆ Architecture & Speed

Zero-Latency Engineering

Most IDE runners experience latency because of standard HTTP buffering and connection handshakes. NYX bypasses these overheads completely using an advanced dual-engine structure.

server/lib/fastifyApi.ts (Performance Architecture) 
// Fastify Socket Configuration to eliminate buffering handshakes
import Fastify from 'fastify';

export const initializeFastifyEngine = () => {
  const fastify = Fastify({ keepAliveTimeout: 75000 });

  // Disable Nagle's algorithm to write packets immediately to TCP sockets
  fastify.addHook('onRoute', (routeOptions) => {
    routeOptions.server.setNoDelay(true);
  });

  // Zero-copy SSE header flushing for instant client stream rendering
  fastify.get('/api/stream', async (request, reply) => {
    reply.raw.writeHead(200, {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache, no-transform',
      'Connection': 'keep-alive'
    });
  });
};
⚡ TCP Nagle Bypass
By explicitly calling setNoDelay(true) on the socket layer, NYX eliminates the 40ms TCP accumulation delay, rendering streaming chunks as soon as the GGUF engine emits them.
🌌 DNS Pre-warming
In the background, NYX pre-warms lookup records for Hugging Face and cloud API endpoints, preventing the first prompt from lagging due to resolution lookup times.
🔒 SHA-256 Prompt Cache
Prompts are hashed and indexed on a lightweight binary local caching folder. Executed commands and repeated requests are served with a 0ms retrieval latency.
